    Primaccess API v1.0

    Scroll down for code samples, example requests and responses. Select a language for code samples from the tabs above or the mobile navigation menu.

    Request format

    Our API only support JSON-encoded data as requests and responses. Headers for both the Content-Type and Accept should be set to application/json.

    Errors

    Occasionally you might encounter errors when accessing the REST API. Error code | Error message ---------- | ------------- 400 Bad Request | Everything worked as expected. 401 Unauthorized | No valid API key provided. 404 Not Found | The requested resource doesn't exist. 500 Internal Server Error | Something went wrong. Errors return both an appropriate HTTP status code and response object which contains a code, message.

    Dates and times

    All dates and times in the API are in UTC. This means that when submitting dates and times in your requests your payload should include date/time in UTC. When processing dates and times in responses from the API you should translate them to the local time of the associated outlet.
